# CS-300 Reflection

## What was the problem you were solving in the projects for this course?

The main problem I was solving throughout this course was determining how to efficiently organize, store, and retrieve data using the appropriate data structures and algorithms. In Project One, I analyzed different data structures to determine which one would provide the best performance for the academic advising software. In Project Two, I applied that analysis by creating a program that could load course information from a file, sort the courses in alphanumeric order, and allow advisors to quickly search for course information and prerequisites.

## How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by first evaluating the strengths and weaknesses of vectors, hash tables, and binary search trees. Understanding data structures is important because the choice of data structure directly impacts the speed, efficiency, and memory usage of a program. Instead of focusing only on making the program work, I had to think about how well it would perform as the amount of data increased. This helped me better understand the relationship between algorithms, run-time complexity, and overall software performance.

## How did you overcome any roadblocks you encountered while going through the activities or project?

One of the biggest challenges was deciding which data structure would best fit the requirements of the advising program while also meeting the need for efficient searching and sorting. Another challenge was working with file input and parsing the comma-separated course data correctly, especially when courses contained different numbers of prerequisites. I overcame these challenges by breaking the problem into smaller pieces, testing individual sections of code, and using debugging techniques to verify that data was being stored and displayed correctly.

## How has your work on this project expanded your approach to designing software and developing programs?

This project helped me shift my focus from simply writing code that works to designing solutions that are efficient and scalable. Before this course, I often focused primarily on functionality. Now, I spend more time thinking about performance considerations, how data will flow through a program, and which data structures will provide the best long-term solution. This experience showed me that good software design starts long before writing the first line of code.

## How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

Working on these projects reinforced the importance of writing clean, organized code that can be easily understood and modified later. I became more intentional about using meaningful variable names, adding comments where appropriate, and separating functionality into smaller, reusable functions. I also gained a better appreciation for designing programs in a way that allows future changes to be implemented with minimal disruption. These practices make programs easier to maintain, improve readability, and increase adaptability as requirements evolve.
